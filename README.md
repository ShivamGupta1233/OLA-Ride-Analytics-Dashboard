# OLA-Ride-Analytics-Dashboard

## 1. Project Title / Headline

### OLA Ride Analytics Dashboard: End-to-End Ride Booking Analysis Using SQL & Power BI

An interactive business intelligence solution designed to analyze OLA ride booking operations, customer behavior, vehicle performance, revenue generation, cancellation patterns, and rating trends through advanced SQL analysis and Power BI visualizations.
## 2. Short Description / Purpose

The OLA Ride Analytics Dashboard is a comprehensive Power BI project built to transform raw ride booking data into actionable business insights. Using SQL for data analysis and Power BI for visualization, the dashboard helps stakeholders monitor booking trends, revenue performance, customer satisfaction, vehicle utilization, and ride cancellation patterns.

This solution is intended for business analysts, operations teams, management executives, and data professionals seeking data-driven insights into ride-hailing operations.
## 3. Tech Stack

The dashboard was built using the following tools and technologies:

* 📊 **Power BI Desktop** – Primary platform for creating interactive dashboards and reports.
* 🗄️ **SQL** – Used for solving business problem statements and extracting meaningful insights from ride booking data.
* 📂 **Power Query** – Data cleaning, transformation, and preparation layer.
* 🧠 **DAX (Data Analysis Expressions)** – Created KPIs, calculated measures, and dynamic business metrics.
* 🔗 **Data Modeling** – Established relationships between booking, customer, vehicle, payment, and rating data.
* 📈 **Data Visualization** – Built interactive charts, KPIs, slicers, and analytical reports.
* 📁 **Excel Dataset (.xlsx)** – Source data for ride booking analysis.
* 📁 **PBIX File (.pbix)** – Dashboard development and reporting environment.

## 5. Features / Highlights

### • Business Problem

Ride-hailing companies generate large volumes of operational data daily. However, extracting meaningful insights regarding customer behavior, revenue generation, vehicle performance, and cancellation patterns can be challenging without proper analytical tools.

Key business questions include:

* Which vehicle categories generate the highest revenue?
* What are the major reasons for ride cancellations?
* Which customers contribute the most bookings?
* How do customer and driver ratings compare?
* Which payment methods are most preferred?
* What trends exist in ride volume over time?

Without visualization and analysis, these questions are difficult to answer efficiently.

### • Goal of the Dashboard

The primary objectives of this dashboard are:

* Analyze ride booking performance.
* Monitor revenue generation and payment trends.
* Track customer and driver satisfaction.
* Identify cancellation patterns and operational issues.
* Evaluate vehicle category performance.
* Support data-driven decision making through interactive reporting.

## SQL Business Problems Solved

The following analytical questions were solved using SQL:

1. Retrieve all successful bookings.
2. Find average ride distance for each vehicle type.
3. Calculate total customer-cancelled rides.
4. Identify top 5 customers with highest bookings.
5. Analyze driver cancellation reasons.
6. Find maximum and minimum driver ratings for Prime Sedan rides.
7. Retrieve all UPI payment transactions.
8. Calculate average customer rating by vehicle type.
9. Compute total booking value of successful rides.
10. Identify incomplete rides and their reasons.
